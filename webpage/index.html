BP

Bluetooth PAN File Transfer & Automated Typing

A lightweight network utility: Phone (Termux) → Laptop (C++ receiver). Reliable framing, atomic saves, hotkey-triggered typing.

Quick Overview

Language: C++ (receiver) + Python (Termux sender) • Platform: Windows + Android (Termux) • Transport: Bluetooth PAN (TCP)

What it does

  • Phone sends file over TCP (Bluetooth PAN) to laptop.
  • Receiver saves payload safely and confirms with ACK.
  • Press 7 + 8 + 9 on laptop to type received text into focused app.

Protocol

Framing: 4-byte big-endian length + payload. Optional 1-byte ACK (0x01).

How to build & run

Compile on Windows (MinGW):

g++ -std=c++17 receiver_win32_fixed.cpp -o receiver.exe -lws2_32

Run receiver:

.\receiver.exe --port 5001 --out "C:\\Users\\YOU\\Desktop\\received_data.txt"

Run Termux sender (on phone):

python3 cn_project_sender.py

Files included

  • receiver_win32_fixed.cpp — C++ receiver (commented)
  • cn_project_sender.py — Python sender for Termux
  • project_report.pdf — Project report
  • demo_video.mp4 — 5 minute demo

Notes

  • Ensure Bluetooth PAN is active and phone & laptop are on same network (e.g., 192.168.44.x).
  • Set SERVER_IP in the sender to the laptop's Bluetooth IPv4 address.
